# HOMEPAGE-PREVIEW.md

## Review Summary
Current homepage base is usable and should be retained.
The issue is not missing content. The issue is weak hierarchy, repeated card logic, and CTA competition.

## What Stays
- Brand banner and brand naming
- Sticky navigation
- Hero core message
- Two core modules
- Seven deliverables content
- Three-phase cooperation process
- Final conversion section

## What Changes
- Hero becomes the clear executive decision section, not a generic landing hero.
- Two-module section becomes the strongest “system architecture” section.
- Deliverables section shifts from equal-weight tiles to grouped operating outputs.
- Process section becomes a more deliberate execution-path section.
- Final CTA becomes a confirmation step, not a repeated sales block.

## What Moves
- System explanation stays below hero, inside the two-module section.
- Process reassurance stays inside the cooperation section, not repeated in the final CTA.
- Any future proof metrics or dashboard-like signals should sit between modules and deliverables, not inside the hero.

## What Gets Simplified
- Banner wording
- Hero support copy
- Visual repetition inside the seven-deliverables grid
- Final CTA messaging

## New Homepage Hierarchy
1. Brand context banner
2. Sticky system navigation
3. Hero: strategic claim + primary action
4. Core modules: what the system is
5. Core deliverables: what gets rebuilt and handed over
6. Cooperation process: how execution unfolds
7. Final CTA: enter the system
8. Footer

## Primary CTA Placement
- Primary page CTA: `进入客户工作台`
- Hero: strongest CTA emphasis
- Navigation: visible but lighter support CTA
- Final CTA: repeat the same primary action after trust has been built
- Secondary CTA: `使用飞书账号登录`

## What The Homepage Should Feel Like After Restructuring
- More like a growth operating system entry page
- Less like a generic B2B marketing site
- More strategic, modular, and high-trust
- More controlled in pacing and clearer in decision hierarchy

## Before vs After
- Before: clear but generic landing-page structure with repeated card patterns
- After: strategic homepage with diagnosis -> system -> deliverables -> process -> action logic

## 3 Most Important Structural Improvements
- Make the two-module section the main structural anchor of the page.
- Reduce equal-weight repetition in the deliverables section.
- Tighten CTA hierarchy so one primary action leads the whole page.

## 3 Biggest Risks If We Keep It As Is
- The page will continue to feel more like a template SaaS site than a ZAPEX operating system.
- Core value will stay flattened into cards instead of reading like a serious system offer.
- Repeated CTA and repeated section patterns will weaken trust and decision clarity.
