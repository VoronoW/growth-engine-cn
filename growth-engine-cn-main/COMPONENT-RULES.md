# COMPONENT-RULES.md

## Buttons
- Primary buttons should feel decisive and high-trust.
- Secondary buttons should remain visible but restrained.
- Button copy should be short, strategic, and action-clear.
- Avoid overly playful shapes or oversized fills.
- Default to one primary button per section and one primary action per page.

## Cards And Modules
- Cards should represent specific functions: module, diagnostic block, phase, proof, or intake section.
- Use clear internal hierarchy: label, title, explanation, action.
- Prefer subtle borders and low-noise surfaces over strong shadows.
- Module blocks should feel more like system panels than marketing tiles.

## Navigation
- Keep global navigation limited and high-signal.
- Labels should map to core decision areas, not filler pages.
- CTA items can be more prominent, but still consistent with the system tone.
- Avoid mixing brochure navigation labels with product-workspace labels in the same row.

## Hero Sections
- Keep heroes concise, high-trust, and structurally clean.
- One strategic claim, one supporting explanation, one or two actions.
- Avoid decorative clutter and avoid overlong value-prop stacking.
- Hero copy should frame the system and outcome, not enumerate every service.

## Content Sections
- Organize sections by logic: diagnosis, system modules, proof, process, conversion.
- Use numbered or labeled structures where they improve executive comprehension.
- Avoid repeating the same card pattern for every section.
- Give each section one primary job: explain, prove, structure, or convert.

## Forms And Intake
- Forms should feel like guided intake into a serious process.
- Use clear step structure, progress visibility, and low-friction wording.
- Inputs should feel calm and professional, not consumer-app friendly.
- Reduce ambiguity in labels, hints, and expected answers.

## Tables And Data Blocks
- Use tables for structured proof, metrics, or diagnostic comparisons.
- Keep them clean, readable, and light in chrome.
- Use numerical emphasis carefully and meaningfully.
- Prefer summary metrics and comparisons over dense raw-data tables on marketing-facing pages.

## Modals And Overlays
- Use modals only for focused confirmation or short guided tasks.
- Avoid modal-heavy workflows when a page section or step flow is clearer.

## Empty States
- Empty states should explain the next operational step.
- Use confident guidance, not playful filler copy.
