# DESIGN.md

## Purpose
This design system defines the company-specific UI direction for `峥锐 ZAPEX｜品牌出海加速器`.
It should feel native to ZAPEX: a premium cross-border growth consulting system with operational clarity.

## Brand Direction
- Premium consulting firm first
- Growth command center second
- SaaS-style operational clarity third
- Strategic, clinical, restrained, high-trust throughout

## Core Product Feeling
- This is not a creative agency site.
- This is not a generic startup landing page.
- This is not a noisy dashboard.
- It should feel like a strategic growth operating system with consulting-grade judgment.

## Visual Principles
- Calm authority over loud persuasion
- System clarity over decorative flourish
- Modular decision surfaces over marketing sections
- Conversion awareness without salesy urgency
- Premium finish without luxury theatrics

## Layout Rules
- Prefer a controlled page frame with clear sectional hierarchy.
- Use modular blocks that feel like diagnostic surfaces, operating modules, or execution stages.
- Keep hero sections clean and decisive, not oversized for spectacle.
- Use strong vertical rhythm with deliberate spacing changes between strategic content, system content, and CTA areas.
- Important sections should read like decision layers: diagnosis, system, execution path, proof, action.
- On most pages, keep section count low and purposeful; each section should answer one executive question.
- Default section order should move from strategic claim -> system explanation -> proof -> process -> action.

## Typography Rules
- Use clean Chinese sans typography with high readability and strong hierarchy.
- Headlines should feel strategic and declarative, not lyrical.
- Body copy should stay concise, dense enough for authority, but never cramped.
- Labels, module headers, and numeric markers should feel operational and precise.
- Avoid overly soft editorial typography and avoid hype-heavy display treatment.
- Use three clear text roles only: strategic headline, explanatory body, operational label.

## Spacing Rhythm
- Use tighter spacing in operational or framework sections.
- Use more breathing room in hero, trust, and high-level narrative sections.
- Preserve visual calm by aligning internal paddings consistently across related modules.
- Avoid both crowded dashboard compression and empty luxury-page spacing.
- Inside a module, prioritize even internal spacing over dramatic visual contrast.

## Color Behavior
- Use a restrained neutral foundation.
- Accent color should support guidance, emphasis, and system status, not carry the whole brand.
- Strong contrast should appear in banners, key indicators, or high-priority CTAs, but remain controlled.
- Avoid excessive gradients, purple-heavy AI styling, or playful consumer palettes.
- Avoid using multiple accent colors on the same screen unless they encode real status meaning.

## Component Character
- Components should feel exact, flat-to-lightly-elevated, and operationally trustworthy.
- Prefer borders, subtle fills, and alignment before shadows.
- Radius should be moderate: modern and refined, not overly soft.
- Cards should read as modules with purpose, not generic containers.
- Buttons should feel direct, confident, and conversion-aware without sounding promotional.

## CTA Hierarchy
- Each page should have one primary CTA only.
- Secondary CTAs may support trust, login, or exploration, but should not compete visually with the primary action.
- In system or diagnostic sections, prefer contextual CTAs over repeated large CTA banners.
- Primary CTA copy should imply the next operational step, not generic enthusiasm.

## Navigation Style
- Navigation should feel like entering a system, not browsing a brochure.
- Keep top-level choices limited and strategic.
- Use labels that imply frameworks, modules, or work surfaces rather than generic marketing tabs.
- Sticky navigation is acceptable if it feels light and disciplined.

## Interaction Tone
- Interactions should feel friction-aware, clear, and reassuring.
- Hover, focus, and active states should be subtle but explicit.
- Avoid playful motion or dramatic animated reveals.
- When forms are involved, the tone should feel like guided intake into a serious process.

## Best Use In This Project
- Homepage: strategic operating-system landing page
- Onboarding: intake workspace / client entry surface
- Dashboard-adjacent areas: diagnostic and execution modules
- CTA areas: confident, calm, high-trust next steps
