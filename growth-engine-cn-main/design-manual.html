Growth OS — The Architectural Ledger

这不是一套营销色板，而是一台高精度仪器的操作档案。在这里我们放弃 SaaS 的圆角与渐变套路， 转向 Bloomberg 终端的信息密度 + 瑞士编辑的克制排印。通过 Intentional Asymmetry 与 Tonal Depth 打破模板感，让界面服务于决策。

02 / NO-LINE

No-Line Rule — 不用 1px 分割界面

把 UI 当作一叠细薄的纸。边界来自 surface 之间的微弱色差， 而不是黑色描边。整套文档本身就在执行这条规则 — 每一个 section 之间都是靠背景切换分开的，没有画过一条线。

04 / DEPTH

Elevation — 由色差构成的层次

放弃 Web-2.0 的下沉阴影。层次来自三种手段：tonal layering（色阶相邻）、 ambient shadow（主动态，ink 4% @ 48px blur），与 ghost border （仅在无障碍需要时启用，outline-variant 15% opacity）。

— RECESSED · surface-highest
凹陷
承载"已完成 / 非活跃"的档案。颜色靠近 surface-dim，看起来像下沉。
○ RESTING · surface-lowest
静息
默认态卡片。坐在 surface-low 工作区之上，靠色差浮起。
⦿ FLOATING · + ambient shadow
浮起（交互中）
只在拖拽、modal、active hover 时启用。48px blur / 4% ink。
04 / RADIUS

06 / FIELD

输入框 — 账本风格底线

只保留底边。surface-low 底色 + outline-variant 50% 底线， focus 时底线变成 primary 2px。这种"账本单行"观感 reinforces the OS metaphor。

11 / RULES

Do / Don't — 可视护栏

违反这四条规则等同于违反"Architectural Ledger"的人格。每一次 UI 评审只看这四条即可快速判断。

DO · 用色差分区

surface 之间的微弱跳层足够作为分界。

拥挤时放大空白，不加边框。纸叠纸的语言 — 这也是整份手册执行的规则。

surface
surface-low
DON'T · 1px 黑描边

不要用 1px solid 把 UI "装进盒子里"。

描边会制造 Web 2.0 的模板感。ghost border 仅在无障碍对比不足时才启用。

← 通用 border · 典型模板感
DO · Manrope 跑数字

任何数字一律走 Manrope tabular-nums。

系统感来自数据对齐。标题与数字用同一套字体传递"同一台机器"的人格。

$ 1,284,930   ·   48.2%
DON'T · Web-2.0 阴影

不要给静态元素加下沉阴影。

阴影只为"正在发生"的动作服务：拖拽、modal、active hover。静息用色差。

错误：下沉 20px / 15% 黑阴影
DON'T · 超大圆角

超过 12px 的圆角会把 OS 变成玩具。

按钮 6px、卡片 8px、hero 12px 已是上限。用矩形承载权威。

错误：pill / 999px 圆角
DON'T · 纯黑 #000

永远用 ink · #1B1C1B 代替 #000。

纯黑在屏幕上反刺眼，破坏"纸上墨水"的质感。所有正文、图标、边界都使用 ink。

错误：背景 / 正文使用 #000
