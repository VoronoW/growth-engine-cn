开发者文档
飞书 OAuth + Bitable API 对接规范
适用接口: 飞书开放平台 v2 · Netlify Functions 服务端代理
架构说明： 本系统采用「飞书 OAuth 2.0 + Netlify Functions 服务端代理」方案。 用户点击登录 → 跳转飞书授权 → 飞书 callback 到 Netlify Function → 写入 session cookie → 前端读取 session → 表单提交走服务端 → 服务端写入飞书多维表格。 在飞书 OAuth 未配置时，表格仍可正常填写并支持导出 CSV 作为兜底。
Netlify Functions 端点
GET /.netlify/functions/feishu-auth-start → 302 跳转飞书授权页 GET /.netlify/functions/feishu-callback → 换 token、写 session、302 回首页 GET /.netlify/functions/feishu-session → 返回当前登录状态与用户信息 POST /.netlify/functions/feishu-submit → body: { panelId, records } → 写入飞书 Bitable
必须在 Netlify 控制台配置的环境变量
FEISHU_APP_ID = cli_xxxxxxxxxxxx FEISHU_APP_SECRET = xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx FEISHU_REDIRECT_URI = https://growth-engine-cn.netlify.app/.netlify/functions/feishu-callback FEISHU_SESSION_SECRET = 任意随机字符串（32位以上） FEISHU_BITABLE_APP_TOKEN = xxxxxxxxxxxxx # 或使用 FEISHU_WIKI_NODE_TOKEN FEISHU_TABLE_A1 = tblxxxxxxxx # 飞书多维表格中 A1 表的 table_id
